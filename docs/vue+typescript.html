Reactive한 데이터 설정

Options API에서는 데이터 속성을 정의함으로써 Reactive한 데이터를 정의할 수 있었습니다.Composition API에서 Reactive한 데이터를 정의하기 위해 reactive 함수, ref 함수 중 하나를 이용합니다.

Reactive란

TypeScript 이전에 Composition API에서의 Reactive한 데이터 이해와 어떻게 다루는지 확인하세요.

App.vue 파일에서 count 변수를 정의하고 click 이벤트에서 count의 수가 1씩 늘어나는 설정을 합니다.


<script setup lang="ts">
const count = 0;
</script>

<template>
  <div>Count:{{ count }}</div>
  <button @click="count++">Add Count</button>
</template>

count 변수를 정의하고 있지만 Reactive한 데이터가 아니기 때문에 count 값은 표시되어도 버튼을 눌러도 count의 수가 늘어나지 않습니다.

ref 함수를 이용하여 Reactive 변수 count를 정의합니다.ref함수 인수에는 count 초기값 0을 설정하고 있습니다.


<script setup lang="ts">
import { ref } from 'vue';

const count = ref(0);
</script>

<template>
  <div>Count:{{ count }}</div>
  <button @click="count++">Add Count</button>
</template>

count는 Reactive 변수이므로 버튼을 클릭하면 count의 수가 늘어납니다.이것이 Reactive 데이터와 Reactive가 아닌 데이터의 차이입니다.

ref 함수의 형 설정

TypeScript 환경에서 Reactive한 데이터가 무엇인지 설명했지만 틀에 대한 설정은 전혀 하지 않았습니다.그것은 TypeScript에 의해 형 추론(Type inference)에 의해 자동으로 형 설정이 이루어지기 때문에 형 설정을 생략할 수 있기 때문입니다.

형 추론에 의해 count 변수에 어떤 형이 설정되어 있는지는 VSCode를 이용하여 확장 기능에 Volar를 설치하고 있다면 count에 커서를 대면 형이 표시됩니다.

count에는 Ref<number>가 설정되어 있는 것을 알 수 있습니다.틀 추론에 따라 틀이 자동으로 설정되어 있기 때문에 click 이벤트에서 count에 문자열을 입력하려고 하면 메시지에서 틀이 다르다는 것을 알려줍니다.

명시적으로 틀을 설정하고 싶은 경우에는 제네릭스를 이용해 설정할 수 있습니다.


const count= ref<number>(0);

이외에 vue에서 Ref형을 import 하여 아래와 같이 설정할 수도 있습니다.


<script setup lang="ts">
import { ref } from 'vue';
import type { Ref } from 'vue';

const count: Ref<number> = ref(0);
</script>

reactive 함수의 형 설정

reactive 함수의 경우는 어떻게 틀을 설정하는지 확인하겠습니다.reactive 함수에서 reactive 변수 user를 정의하여 FullName으로 브라우저 상에 이름을 표시하도록 하고 있습니다.


<script setup lang="ts">
import { reactive } from 'vue';
const user = reactive({
  firstName: 'John',
  lastName: 'Doe',
  age: 25,
});
</script>

<template>
  <div>FullName: {{ `${user.firstName} ${user.lastName}` }}</div>
</template>

reactive 함수를 설정할 때 틀을 설정하지 않았지만 틀 추론에 의해 틀을 확인할 수 있습니다.

명시적으로 틀을 설정하고 싶다면 아래와 같이 정의한 변수 user 오른쪽에 :(콜론)을 붙여 설정할 수 있습니다.객체 기술 방식과 비슷하지만 속성과 틀 쌍을 여러 개 설정할 때 콤마를 이용하지 않으므로 주의하십시오.틀을 지정할 수 있었지만 오브젝트의 속성의 수가 늘어나면 속성명이나 틀 설정의 잘못을 일으키기 쉬워 코드도 읽기 어려워집니다.


const user: {
  firstName: string;
  lastName: string;
  age: number;
} = reactive({
  firstName: 'John',
  lastName: 'Doe',
  age: 25,
});

interface를 이용하면 틀을 따로 정의할 수 있어 코드가 깔끔하고 읽기 쉬워집니다.또한 interface는 import, export 할 수 있기 때문에 한 번 어딘가에서 정의하는 것으로 틀의 재사용을 실시할 수 있습니다.정의 장소를 하나로 함으로써 매번 틀을 기술함으로써 생기는 오류를 없앨 수 있습니다.


interface User {
  firstName: string;
  lastName: string;
  age: number;
}

const user: User = reactive({
  firstName: 'John',
  lastName: 'Doe',
  age: 25,
});

오브젝트의 틀을 설정할 수 있는 것은 interface 뿐만 아니라 type(틀 에일리어스)을 이용할 수 있습니다.interface와 type은 기술 방법은 비슷하지만 type에는 =(이콜)을 이용하여 설정하는 등 차이가 있으므로 주의가 필요합니다.


type User = {
  firstName: string;
  lastName: string;
  age: number;
};

const user: User = reactive({
  firstName: 'John',
  lastName: 'Doe',
  age: 25,
});

기타 interface, type과 TypeScript의 기본적인 기능에 대해서는 아래 문서에서 공개하고 있습니다.

형 어서션

user를 정의할 때 초기값이 빈 객체이고 나중에 코드 안에서 속성을 설정하는 경우도 있을 수 있습니다.


const user = reactive({})

user.firstName="John",
user.lastName="Doe",
user.age = 25;

틀 추론에 의해 user의 틀은 {}로 되어 있기 때문에 속성에 값을 설정하려고 하면 오류 메시지가 나타납니다.

그 경우에 형 어서션을 이용하는 것으로 형을 덮어쓸 수 있습니다.


const user = reactive({} as User)

user.firstName="John",
user.lastName="Doe",
user.age = 25;

computed 속성 유형 설정

computed 속성에 있어서 틀 설정에 대해 확인을 실시합니다.computed 속성은 정의된 변수를 이용하여 계산, 가공을 함으로써 원래 변수의 데이터와 다르게 사용자에게 표시할 수 있는 기능입니다.reactive 변수를 이용하여 computed 속성을 정의한 경우 reactive 변수가 갱신되면 그 갱신에 맞추어 재계산, 재가공이 자동으로 이루어집니다.

Composition API에서 computed 속성을 이용하기 위해서는 vue에서 computed 함수를 import 해야 합니다.computed 속성에서는 computed 함수 인수에 함수를 설정합니다.동작 확인을 위해 computed 속성을 설정하는데 여기서 설정하는 computed 속성은 reactive 함수에서 정의한 user 변수를 이용하여 firstName과 lastName을 결합하여 표시하기만 하면 되는 간단한 것입니다.


import { reactive, computed } from 'vue';

type User = {
  firstName: string;
  lastName: string;
  age: number;
};

const user: User = reactive({
  firstName: 'John',
  lastName: 'Doe',
  age: 25,
});

const fullName = computed(() =>`${user.firstName} ${user.lastName}`);

fullName에 커서를 대면 형 추론에 의해 fullName의 형이 자동으로 설정되어 ComputedRef<string>임을 알 수 있습니다.

명시적으로 틀을 지정하고 싶은 경우는 ref와 같이 제네릭스를 이용해 설정을 실시할 수 있습니다.Computed 속성 인수로 설정한 함수의 반환 값 유형을 설정합니다.


const fullName = computed<string>(() => `${user.firstName} ${user.lastName}`);

함수의 형 설정

Vue의 Options API에서는 데이터 속성을 업데이트할 때 methods(메소드)에 함수를 추가했지만 Compotions API에서는 script 태그 안에 그대로 함수를 기술하는 것이 reactive 변수를 업데이트 할 수 있습니다.TypeScript에서의 함수 틀 설정과 방법은 동일합니다.

changeName을 인수 없이 정의하고 user.firstName 값을 "Jane"으로 설정하여 실행합니다.


const changeName = () => {
  user.firstName = "Jane";
};

changeName();

브라우저 상에는 "Jane Doe"라고 표시되며 함수의 처리는 정상적으로 동작합니다.

함수의 경우 인수와 반환 값에 대해 틀을 설정할 수 있습니다.반환값의 설정을 실시하지 않은 경우에도 형 추론에 의해서 자동으로 형이 설정됩니다.

changeName 함수에는 반환 값이 없기 때문에 반환 값의 형이 void 형으로 되어 있음을 알 수 있습니다.반환 값의 틀을 명시적으로 지정하고자 하는 경우에는 아래와 같이 설정할 수 있습니다.


const changeName = ():void => {
  user.firstName = "Jane";
};

그런 다음 함수에 인수를 설정합니다.


const changeName = (name):void => {
  user.firstName = name;
};

changeName('Jane');

인수로 name을 지정하고 있지만 틀 지정이 없기 때문에 VSCode 상에서는 name 아래에 파선이 표시됩니다.메시지가 표시됩니다.name에 커서를 맞추면 메시지에는 "name은 암묵적으로 any 타입을 가지고 있다"고 표시됩니다.

npm run dev 명령어 상에는 오류 메시지가 발생하지 않았지만 TypeScript 유형 체크를 수행할 수 있는 npm runtypeck 명령어를 실행해 보겠습니다. npm runtypecheck 명령어를 실행할 수 있다는 것은 package.json 파일의 scripts에 typecheck이 등록되어 있는 것으로 확인할 수 있습니다.실행하면 VSCode 상에 표시되어 있던 에러와 같은 내용이 표시됩니다.


 % npm run typecheck

> vue3-typescript@0.0.0 typecheck
> vue-tsc --noEmit

src/App.vue:18:21 - error TS7006: Parameter 'name' implicitly has an 'any' type.

18 const changeName = (name): void => {

메시지를 보면 알 수 있듯이 암묵적으로 any형이 설정되어 있는 것이 원인이 되어 에러가 나고 있습니다.암묵적으로 any형이 설정되어 있다고 반드시 오류가 나는 것은 아니며 TypeSciprt의 컴파일러 옵션의 no ImplicitAny에서 제어할 수 있습니다.no Implicit Any라는 이름에서 implicit(암묵적)인 any형은 허용하지 않는다는 것을 나타냅니다.이것은 tsconfig.js 파일에서 strict가 true로 되어 있기 때문에 no Implicit Any가 활성화 되어 있습니다.tsconfig.js 파일에서 strict 값을 false로 설정하면 암묵적인 any 가 허용되며 오류는 사라집니다.


{
  "extends": "@vue/tsconfig/tsconfig.web.json",
  "include": ["env.d.ts", "src/**/*", "src/**/*.vue"],
  "compilerOptions": {
    "strict": false,//試しに設定
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  //略

strict 값을 true로 설정하면 no Implicit Any를 포함한 7가지 옵션이 활성화됩니다.strict라는 이름 그대로 설정을 하면 틀 체크가 엄격해집니다.

strict를 false로 변경한 후 npm runtypecheck 명령을 실행하면 오류 메시지가 사라집니다.

tsconfig.json 설정값에 대해서는 extends로 설정되어 있는 @vue/tsconfig/tsconfig.web.json을 보면 @vue/tsconfig/의 tsconfig.json으로 설정되어 있는 것을 알 수 있습니다.

strict를 tsconfig.json에서 true로 설정하고 있다면 함수 인수에는 반드시 틀을 지정해야 한다는 것을 알 수 있습니다.

tsconfig.json 파일을 업데이트하면 설정값이 반영되는 것을 확인할 수 있었으므로 strict는 true로 되돌려 놓으십시오.any형을 이용하면 어떤 값이든 넣을 수 있기 때문에 코드의 품질이 떨어져 버리기 때문에 strict는 true로 설정해 둡니다.

함수의 반환 값에 대해서는 형 추론으로 형이 설정되어 있기 때문에 명시적으로 설정할 필요는 없었지만 인수의 경우 명시적으로 형을 설정할 필요가 있습니다.


const changeName = (name:string): void => {
  user.firstName = name;
};

인수, 반환 값의 틀을 설정했지만 함수에 대해서도 틀을 설정할 수 있습니다.애로우 함수와 매우 유사하지만 아래와 같이 틀을 설정할 수 있습니다.(name:string) =>void 부분이 함수로 설정한 틀을 나타냅니다.


const changeName: (name: string) => void = (name: string): void => {
  user.firstName = name;
};

Props의 형 설정

props는 부모와 자녀 관계를 가진 컴포넌트에서 데이터를 주고받기 위해 이용하는 기능입니다.동작 확인을 위해서는 다른 컴포넌트를 작성해야 합니다.기본적으로 존재하는 src¥components 폴더에 있는 Hello World.vue 파일을 이용합니다.

Hello World.vue에서는 props의 설정이 이루어지고 있으므로 설정되어 있는 props를 이용해 이하와 같이 기술합니다.


<script setup lang="ts">
defineProps<{
  msg: string;
}>();
</script>

<template>
  <div>
    <h1>{{ msg }}</h1>
  </div>
</template>

App.vue 파일로 작성한 Hello World.vue 파일을 import하여 props로 msg을 전달합니다.


<script setup lang="ts">
import HelloWorld from "./components/HelloWorld.vue";
</script>

<template><HelloWorld msg="Hello" /></template>

브라우저 상에는 Hello가 표시됩니다.

"script setup" 과 Composition API 에서 props 를 이용할 경우에는 define Props 함수를 이용합니다.ref나 computed와 달리 defineProps 함수를 import 할 필요가 없습니다.defineProps 함수는 컴파일러 매크로라고 불리며 script setup 안에서 이용할 수 있습니다.그 외 컴파일러 매크로에는 나중에 설명할 emit에서 이용되는 defineEmit 함수가 있습니다.

상기 Hello World 컴포넌트의 설정에서 확인한 바와 같이 props로 전달되는 변수는 <> 안에서 틀을 설정할 수 있습니다.

복수의 props가 있는 경우의 설정 방법을 확인합니다.ref함수를 이용하여 정의한 user를 props로 전달합니다.


<script setup lang="ts">
import HelloWorld from "./components/HelloWorld.vue";
import { ref } from "vue";

export type User = {
  firstName: string;
  lastName: string;
  age: number;
};

const user = ref<User>({
  firstName: "John",
  lastName: "Doe",
  age: 25,
});
</script>

<template><HelloWorld msg="Hello" :user="user" /></template>

type(형 에일리어스)이나 interface 등으로 설정한 형은 export하여 다른 컴포넌트로 이용할 수 있습니다.User형도 Hello World 컴포넌트에서 이용할 수 있도록 export를 부착하고 있습니다.export를 이용해 틀을 재사용할 수 있는 것으로 오브젝트와 같은 기술량이 많은 틀을 매번 모두 기술할 필요가 없어집니다.

Hello World.vue 파일에서는 defineProps를 사용하여 props의 msg와 user의 틀을 설정합니다.User형은 App.vue 파일에서 import 하고 있습니다.


<script setup lang="ts">
import type { User } from "../App.vue";
defineProps<{
  msg: string;
  user: User;
}>();
</script>

<template>
  <div>
    <h1>{{ msg }}</h1>
    <div>fullName: {{ `${user.firstName} ${user.lastName}` }}</div>
  </div>
</template>

복수의 props가 있는 경우는 상기와 같이 설정을 실시할 수 있습니다.Props라는 이름의 interface로 정리하고 있습니다.


<script setup lang="ts">
import type { User } from "../App.vue";
interface Props {
  msg: string;
  user: User;
}
defineProps<Props>();
</script>

<template>
  <div>
    <h1>{{ msg }}</h1>
    <div>fullName: {{ `${user.firstName} ${user.lastName}` }}</div>
  </div>
</template>

Emit의 형 설정

부모 컴포넌트에서 자녀 컴포넌트로 값을 전달하고자 하는 경우에 props를 이용할 수 있습니다.그 반대로 자녀 컴포넌트에서 부모 컴포넌트에 대해 통지 또는 값을 전달하고자 할 때 이용할 수 있는 것이 emit입니다.

Hello World.vue 파일에 버튼을 추가하고 click 이벤트를 이용하여 changeName 함수를 설정합니다.버튼을 클릭하면 changeName 함수가 실행됩니다.props를 통해 user 객체가 전달되고 있지만 props로 전달된 데이터를 하위 컴포넌트로 업데이트할 수 없습니다.


<script setup lang="ts">
import type { User } from "../App.vue";
interface Props {
  msg: string;
  user: User;
}
defineProps<Props>();

const changeName = () => {
  console.log("change name");
};
</script>

<template>
  <div>
    <h1>{{ msg }}</h1>
    <div>fullName: {{ `${user.firstName} ${user.lastName}` }}</div>
    <button @click="changeName">Change Name</button>
  </div>

user 객체에 포함된 속성을 업데이트하려면 부모 컴포넌트인 App.vue 파일로 수행해야 합니다.emit을 이용하여 부모 컴포넌트에 대해 변경하고자 하는 값을 전달할 수 있습니다.

emit을 이용하기 위해서는 defineEmits 함수를 이용하여 emit 설정을 해야 합니다.defineEmits 인수에는 함수형 정의(콜 시그니처)를 기술합니다.event는 emit을 실행할 때 설정할 이벤트 이름을 설정합니다.이 이벤트 이름을 이용하여 부모 컴포넌트에서 자녀 컴포넌트에서 보내오는 이벤트를 식별할 수 있습니다.두 번째 인수에는 부모 컴포넌트에 전달할 변수(payload: 페이로드)의 틀을 설정합니다.함수의 반환 값은 없으므로 void형을 설정하고 있습니다.넘겨주기 값이 없을 경우 제2인수는 생략할 수 있습니다.


const emit = defineEmits<{
  (event: "changeName", firstName: string): void;
}>();

emit에서 설정한 정의를 바탕으로 changeName 함수에서 실행하는 emit의 처리를 기술합니다.이벤트 이름에는 defineEmits에서 설정한 changeName을 설정하고 부모 컴포넌트에는 문자열(string) Jane을 전달합니다.이제 자컴포넌트측emit설정이완료되었습니다.


const changeName = () => {
  emit("changeName", "Jane");
};

자녀 컴포넌트 설정이 완료되면 부모 컴포넌트에서 changeName 이벤트를 탐지하는 설정을 합니다.


<template>
  <HelloWorld msg="Hello" :user="user" @changeName="changeName" />
</template>

changeName 이벤트 탐지는 @changeName에서 수행할 수 있습니다.changeName 이벤트를 감지하면 changeName 함수를 실행합니다.이벤트 이름과 함수 이름을 같게 하고 있지만 함수 이름을 변경하는 것은 가능합니다.changeName 인수에는 emit에서 전달한 값이 들어가기 때문에 틀을 같은 string으로 설정합니다.


const changeName = (firstName: string) => {
  user.value.firstName = firstName;
};

브라우저에서 동작확인을 하면 "ChangeName" 버튼을 클릭하면 브라우저 상에 표시된 "John Doe"가 "Jane Doe"로 바뀝니다.

reactive한 값을 주다

부모 컴포넌트에 전달하던 값이 "Jane"으로 설정되어 있었으므로 고정값이 아닌 입력한 값을 전달할 수 있도록 input 요소를 추가합니다.input 요소에 입력한 값을 가져올 수 있도록 v-model 디렉티브에서 firstName을 지정합니다.firstName은 reactive 변수로 ref 함수를 이용하여 정의합니다.


<script setup lang="ts">
import { ref } from "vue";
import type { User } from "../App.vue";
interface Props {
  msg: string;
  user: User;
}
defineProps<Props>();

const emit = defineEmits<{
  (event: "changeName", firstName: string): void;
}>();

const changeName = () => {
  emit("changeName", firstName.value);
};

const firstName = ref<string>("");
</script>

<template>
  <div>
    <h1>{{ msg }}</h1>
    <div>fullName: {{ `${user.firstName} ${user.lastName}` }}</div>
    <input v-model="firstName" /><br />
    <button @click="changeName">Change Name</button>
  </div>

firstName의 초기값은 ""로 틀은 string으로 설정하고 있습니다.부모 컴포넌트에 전달하는 값은 ref 함수이므로 firstName.value 형태로 전달하고 있습니다.

input 요소에 임의의 이름을 입력하고 "ChangeName" 버튼을 클릭하면 이름이 업데이트됨을 확인할 수 있습니다.

ref함수에서 정의한 firstName을 전달할 때 firstName.value로 전달했었는데 firstName으로 전달할 수 있는지 확인합니다.

emit 함수의 제2 인수를 firstName으로 변경하면 틀이 Ref<string>으로 바뀌기 때문에 메시지가 표시됩니다.메시지가 표시되는 것은 defineEmits 안에서 설정한 firstName의 틀이 string이기 때문에 틀이 일치하지 않기 때문입니다.

firstName의 틀을 string에서 Ref<string>으로 변경합니다.Ref를 이용할 경우 Ref를 vue에서 import해야 합니다.


import { ref } from "vue";
import type { Ref } from "vue";
//略

const emit = defineEmits<{
  (event: "changeName", firstName: Ref<string>): void;
}>();

Hello World.vue 파일을 업데이트 했으므로 App.vue로 메시지가 표시됩니다.defineEmits에서 함수의 틀을 변경하였으므로 App.vue측의 changeName 인수의 틀을 일치시켜야 합니다.

firstName의 틀을 string에서 Ref<string>으로 변경합니다.Ref의 import도 필요합니다.값은 fistName.value에서 얻을 수 있습니다.


import HelloWorld from "./components/HelloWorld.vue";
import { ref } from "vue";
import type { Ref } from "vue";
//略
const changeName = (firstName: Ref<string>) => {
  user.value.firstName = firstName.value;
};

틀을 올바르게 설정함으로써 emit에서 reactive 변수를 보낼 때는 두 가지 방법이 있음을 알 수 있었습니다.

emit의 다른 기술 방법

defineEmits 인수에서는 함수의 틀을 설정하고 있었지만 이벤트명만 설정할 수도 있습니다.


const emit = defineEmits(["changeName"]);

이벤트명이 일치하므로 위 설정에서 emit을 실행할 수 있지만 틀 정의를 하지 않았기 때문에 인수에 수치로 해도 VScode 상에서 메시지로 알려주지는 않습니다.

TemplateRefs형태설정

Vue.js에서는 DOM 요소에 직접 접근하고 싶은 경우에도 ref 함수를 이용할 수 있습니다.Template Refs에서 ref 함수를 이용한 경우 틀 설정을 확인합니다.

App.vue 파일에서 ref를 이용하여 input 요소에 접근할 수 있도록 아래를 기술합니다.액세스하려는 요소에 ref 속성을 설정하여 임의의 이름을 설정합니다.ref 함수를 이용하여 정의하는 변수는 요소의 ref 속성에서 지정한 이름과 동일한 이름을 설정해야 합니다.


<script setup lang="ts">
import { ref,onMounted } from "vue";

const input = ref(null);

onMounted(() => {
  input.value.focus();
});
</script>

<template>
  <input ref="input" />
</template>

위 코드는 자바스크립트에서는 문제가 없지만 TypeScript에서 이용하면 input.value 아래에 물결선이 표시되고 메시지가 표시됩니다.메시지 내용은 "객체는 'null'" 일 가능성이 있습니다.ts(2531)"입니다.원인은 초기값에 null을 설정하고 있기 때문입니다.틀을 명시하지 않고 메시지를 해소하기 위해 ref 함수의 초기값을 null에서 공백으로 변경합니다.


const input = ref();

onMounted(() => {
  input.value.focus();
});

input 변수에 커서를 대면 틀을 확인할 수 있습니다.Ref<any>로 되어 있어 제네릭스에는 any형이 설정되어 있습니다.any형은 어떤 값이든 이용할 수 있기 때문에 메시지가 해소됩니다.

변수 input 안에 input 요소의 정보가 포함되어 있는지 확인하기 위해 input.value의 내용물을 console.log로 지정합니다.


const input = ref();

onMounted(() => {
  console.log(input.value);
});

브라우저의 디벨로퍼툴 콘솔에는 <input />가 표시되며 변수 input 안에 input 요소가 포함되어 있음을 확인할 수 있습니다.

input 요소는 focus 메서드, value 속성을 가지고 있으므로 아래와 같이 설정하면 브라우저에서 표시했을 때 해당 요소에 포커스가 맞춰져 value 값을 디벨로퍼 도구 콘솔에 표시할 수 있습니다.


<script setup lang="ts">
import { ref, onMounted } from "vue";

const name = ref<string>("John");
const input = ref();

onMounted(() => {
  input.value.focus();
  console.log(input.value.value);
});
</script>

<template>
  <input ref="input" v-model="name" />
</template>

브라우저의 개발자 도구 콘솔에는 "John"이 표시됩니다.any형이 설정되어 있기 때문에 특별히 틀에 관한 메시지가 표시되지 않고 동작하는 것을 알 수 있었습니다.

TypeScript에서는 DOM 요소에 틀을 설정할 수 있기 때문에 any형이 아닌 명시적으로 틀을 설정해 보도록 하겠습니다.input 요소의 틀은 HTML input Element이므로 HTML input Element를 설정합니다.


const input = ref<HTMLInputElement>();

HTMLInputElement를 설정하면 input변수의 틀은 형추론에 따라 Ref <HTMLInputElement | undefined>가 되고 input의 값이 undifined인 경우 input.value는 값이 없기 때문에 아래 메시지가 표시됩니다.

input.value가 값을 갖는지 체크하면 메시지가 해소됩니다.


onMounted(() => {
  if (input.value) {
    input.value.focus();
    console.log(input.value.value);
  }
});

if문에 의한 분기가 아닌 optional chainning을 이용해도 메시지는 해소됩니다.


onMounted(() => {
    input.value?.focus();
    console.log(input.value?.value);
});

union형을 이용하여 undefined도 명시적으로 설정할 수 있습니다.


const input = ref<HTMLInputElement | undefined>();


const input = ref<HTMLInputElement | undefined>();

ref함수의 값이 공백이 아닌 처음 설정했던 초기값의 null로 했을 경우도 아래와 같이 기술할 수 있습니다.


const input = ref<HTMLInputElement | null>(null);

HTMLInputElement유형확인

input 요소의 틀에 갑자기 HTMLInputElement가 나오는데 HTMLInputElement의 틀을 구체적으로 확인하는 것은 인터넷상에서 검색할 수도 있지만 VSCode의 경우 HTMLInputElement에 커서를 맞추고 "Ctrl + 클릭"하면 틀 세부사항을 확인할 수 있습니다.

HTMLInputElement는 HTMLElement를 exteds(계승)하고 있는 것도 알 수 있습니다.

스크롤을 하다 보면 value 속성을 찾을 수 있고 string형이 설정되어 있음을 알 수 있습니다.

input의 틀을 HTMLElement로 변경하면 HTMLElement의 틀은 value 속성을 가지고 있지 않기 때문에 메시지가 표시됩니다.


const input = ref<HTMLElement | null>(null);

HTMLElement는 HTMLOrSVGElement를 exteds(계승)하고 있으므로 focus 메서드는 가지고 있기 때문에 input.value 행을 삭제하면 메시지가 사라집니다.

template refs를 이용하여 직접 요소에 접근하는 경우 명시적으로 틀을 설정하는 경우 접근할 요소가 어떤 요소인지 확인하여 올바른 틀을 설정해야 합니다.

예를 들어 a태그라면 HTMLAnchorElement의 틀을 설정하는 것으로 href에의 액세스를 실시해도 메시지가 표시되지 않습니다.HTMLElement나 HTMLInputElement의 틀을 설정하면 속성 href는 HTMLElement에 존재하지 않는다고 나옵니다.


<script setup lang="ts">
import { ref, onMounted } from "vue";

const name = ref<string>("John");
const input = ref<HTMLElement | null>(null);
const ahref = ref<HTMLAnchorElement | null>(null);

onMounted(() => {
  input.value?.focus();
  console.log(ahref.value?.href);
});
</script>

<template>
  <input ref="input" v-model="name" />
  <a ref="ahref" href="https://google.com">Google</a>
</template>

template refs에서의 형 설정을 통해 HTMLElement의 형 기초도 이해할 수 있게 되었습니다.

Event의 형 설정

Event에 관한 형 설정에 대해 확인합니다.input 요소를 준비하여 change 이벤트를 설정합니다.input 요소에 입력하고 커서를 제거하면 handle Change 함수가 실행됩니다.


<script setup lang="ts">
const handleChange = (event) => {
  console.log(event.target.value);
};
</script>

<template>
  <input @change="handleChange" />
</template>

상기 코드의 경우 인수에 틀을 설정하지 않았기 때문에 메시지가 표시됩니다.함수일 때 확인했는데 인수에 틀을 설정하지 않은 경우 any형이 설정되는데 TypeScript 설정 파일의 tsconfig.json에서 strict를 true로 설정했기 때문에 "noImplicitAny"가 활성화되어 암묵적인 any는 이용할 수 없습니다.

event에는 Event형이 있으므로 설정합니다.Event형을 지정하면 앞으로는 event.target.value 아래에 파선이 표시됩니다.메시지를 확인하면 객체가 null일 수 있다는 것입니다.

event의 target에 무엇이 들어있는지 확인해보겠습니다.


const handleChange = (event: Event) => {
  console.log(event.target);
};

브라우저의 디벨로퍼 툴의 콘솔에는<input/>가 들어가 있는 것을 알 수 있습니다.input 요소의 틀은 template Refs 안에서 확인한 바와 같이 HTML Input Element였음을 기억해 보세요.

change 이벤트를 실행하는 요소는 반드시 input 요소임을 알고 있으므로 event.target에는 반드시 input 요소가 들어갑니다.null을 피하기 위해서 event.target의 틀을 형어서션을 사용해서 HTMLInputElement의 틀을 덮어씁니다.


<script setup lang="ts">
const handleChange = (event: Event) => {
  console.log((event.target as HTMLInputElement).value);
};
</script>

<template>
  <input @change="handleChange" />
</template>

메시지는 사라지고 input 요소에 문자열을 입력하고 커서를 제거하면 입력한 문자열이 개발자 도구 콘솔에 표시됩니다.

인수가 있을 경우

handle Change 함수에 인수가 있을 경우도 확인해 둡니다.아래와 같이 handle Change에 $event를 이용하면 인수와 별도로 이벤트를 얻을 수 있습니다.


<script setup lang="ts">
const handleChange = (event: Event, comment: string) => {
  console.log((event.target as HTMLInputElement).value);
　　　　//const target = event.target as HTMLInputElement
  //console.log(target.value)
  console.log(comment);
};
</script>

<template>
  <input @change="handleChange($event, 'eventの型の確認')" />
</template>

mousemove 이벤트의 예

mousemove 이벤트를 사용하여 마우스 위치 정보를 가져오는 코드를 이용하여 Event의 형태를 확인합니다.


<script setup lang="ts">
import { ref, onMounted, onUnmounted } from "vue";

const x = ref<number>(0);
const y = ref<number>(0);

const handleMouseMove = (event: Event) => {
  x.value = event.pageX;
  y.value = event.pageY;
};

onMounted(() => {
  window.addEventListener("mousemove", handleMouseMove);
});

onUnmounted(() => {
  window.removeEventListener("mousemove", handleMouseMove);
});
</script>

<template>
  <div>
    <span>x:{{ x }} y:{{ y }}</span>
  </div>
</template>

event에 Event형을 설정하고 있기 때문에 이것으로 문제가 없다고 생각하실 수 있지만 pageX와 pageY에 물결선이 표시됩니다.메시지에는 Event형은 속성 X를 가지고 있지 않다 이런 내용입니다.VScode라면 Event에 커서를 대고 "Ctrl + 클릭"을 하여 Event의 내용물을 확인합니다.속성의 target이나 type 등을 찾을 수 있지만 pageX, pageY는 없습니다.

HTMLElement를 상속받던 HTMLInputElement나 HTMLDivElement가 있듯이 Event를 상속하는 MouseEvent형이 존재합니다.MouseEvent형을 설정하면 메시지 표시가 사라집니다.


const handleMouseMove = (event: MouseEvent) => {
  x.value = event.pageX;
  y.value = event.pageY;
};
</script>

MouseEvent형 내용물을 보시면 pageX, pageY를 확인하실 수 있습니다.UIEvent를extends(계승)하고있는데UIEvent는Event를extends하고있습니다.

Event라고 해도 다양한 틀이 존재하기 때문에 이벤트에 맞는 틀을 선택해야 합니다.

fetch 함수의 형 설정

외부 리소스에서 fetch 함수를 이용해 취득한 데이터를 v-for 디렉티브를 이용해 전개 표시할 경우의 설정을 확인해 둡니다.외부 리소스로는 JSONPlaceholer를 이용합니다.무료 서비스로 https://jsonplaceholder.typicode.com/users 에 접속하면 10명분의 사용자 정보를 얻을 수 있습니다.

TypeScript를 이용하지 않는 경우는 아래와 같이 기술하는 것으로 브라우저 상에 10명분의 사용자명을 표시할 수 있습니다.취득할 때 fetch 함수와 async, await 함수를 이용하고 있습니다.


<script setup lang="ts">
import { ref } from "vue";
const users = ref([]);

const fetchUsers = async () => {
  const res = await fetch("https://jsonplaceholder.typicode.com/users");
  users.value = await res.json();
};

fetchUsers();
</script>

<template>
  <div v-for="user in users" :key="user.id">
    {{ user.name }}
  </div>
</template>

이 코드를 그대로 TypeScript로 이용하려면 v-for디렉티브의 key의 user.id 에서 "속성 'id' 하형 'never' 에 존재하지 않습니다."라고 표시됩니다.user.name 도 같은 메시지가 표시됩니다.

변수 users에 커서를 맞추면 Ref<never[]로 배열의 never형이 설정되어 있을 수 있습니다.

ref함수에 명시적으로 틀을 설정합니다.JSON Place Holder에서 반환되는 사용자 정보를 확인해야 합니다.직접 브라우저 URL에 https://jsonplaceholder.typicode.com/users 을 입력하면 사용자 정보를 얻을 수 있습니다.

interface User {

interface의 User를 작성 후 ref함수에 틀을 설정합니다.본 문서에서 설명이 끝났지만 ref함수는 2가지 방법으로 틀을 설정할 수 있습니다.


const users = ref<User[]>([]);

const users: Ref<User[]> = ref([]);

ref함수에 틀을 설정하면 user.id 의 메시지가 해소되고 브라우저 상에는 취득한 사용자 일람이 표시됩니다.

fetchUsers에 커서를 대면 형 추론에 의해 함수의 반환 값이 Promise<void>인 것을 알 수 있습니다.async의 반환 값은 Promise의 제네릭스에 반환 값의 틀을 넣는 것을 알 수 있습니다.

명시적으로 틀을 지정하고 싶은 경우는 아래와 같이 실시할 수 있습니다.반환 값이 없기 때문에 void형을 지정하고 있습니다.


const fetchUsers = async (): Promise<void> => {
  const res = await fetch(
    "https://jsonplaceholder.typicode.com/users"
  );
  users.value = await res.json();
};

만약 users.value와 같이 값을 되돌리는 경우는 void형이 아닌 되돌려지는 형(User[])을 설정하게 됩니다.


const fetchUsers = async (): Promise<User[]> => {
  const res = await fetch(
    "https://jsonplaceholder.typicode.com/users"
  );
  users.value = await res.json();
  return users.value;
};

다음은 await에서 실행되는 fecth 함수의 반환 값을 확인하기 위해 변수의 res에 커서를 맞춥니다.

형 추론에 의해 Response 형이 설정되어 있는 것을 알 수 있습니다.Response의 틀이란 어떤 것일까요?

명시적으로 반환 값에 Response 형을 설정하여 형의 내용물을 확인해 봅시다.res에 Response 형을 설정한 후 VSCode의 경우 Response에 커서를 대고 "Ctrl + 클릭"으로 형 정보가 표시됩니다.


const res:Response= await fetch("https://jsonplaceholder.typicode.com/users");

interface Response 틀은 아래와 같이 설정되어 있습니다.Response는 body를 extends(계승)하고 있기 때문에 Body형 내용물도 함께 확인해 보도록 하겠습니다.


interface Response extends Body {
    readonly headers: Headers;
    readonly ok: boolean;
    readonly redirected: boolean;
    readonly status: number;
    readonly statusText: string;
    readonly type: ResponseType;
    readonly url: string;
    clone(): Response;
}
//Boty
interface Body {
    readonly body: ReadableStream<Uint8Array> | null;
    readonly bodyUsed: boolean;
    arrayBuffer(): Promise<ArrayBuffer>;
    blob(): Promise<Blob>;
    formData(): Promise<FormData>;
    json(): Promise<any>;
    text(): Promise<string>;
}

interface를 구성하는 속성 중에는 status나 ok 등 익숙한 것도 있지 않을까요?익숙하지 않은 사람도 res의 내용물을 console.log로 출력함으로써 Reponse의 속성과 변수 res에 포함된 속성의 정보가 일치한다는 것을 이해할 수 있습니다.


const fetchUsers = async (): Promise<User[]> => {
  const res: Response = await fetch(
    "https://jsonplaceholder.typicode.com/users"
  );
  console.log(res);
  users.value = await res.json();
  return users.value;
};

브라우저 개발자 도구의 콘솔을 확인합니다.interface의 Reponse에 정의된 속성과 res에 포함된 속성이 일치함을 알 수 있습니다.

실제 값과 틀을 비교함으로써 Response 형에 대한 이해도 깊어진 것 같습니다.취득한 res 객체로부터 사용자 데이터를 취득하는 json 메서드는 Interface의 Body형에 포함되어 있으므로 반환 값이 Promise<any>인 것도 알 수 있습니다.


users.value = await res.json();

이렇게 Response나 HTMLElement 등 스스로 설정하지 않은 틀을 확인해 나가면서 그 틀을 설정한 변수들이 어떤 속성이나 메서드를 가지고 있는지 확인할 수 있습니다.틀을 이해할 수 있으면 어떤 기능을 가지고 있는지도 이해할 수 있게 됩니다.
